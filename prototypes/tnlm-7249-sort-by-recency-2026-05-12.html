TNLM-7249 — Sort By: Add "Most Recently Updated/Listed"

Surface

OneBid portal · /main/search · Sort By dropdown (top-left of results toolbar, inside the search results header).

Change

One new option added to the Sort By dropdown at position 1 (above "Bid Close Date"). All other rows unchanged. Label: Most Recently Updated/Listed.

Interaction flow

  • 1 · Default — Sort trigger collapsed (icon + red chevron, no active state).
  • 2 · Sort Open — Click trigger. Popup opens. Trigger gets red border. New row at position 1 followed by the three existing rows.
  • 3 · Selected — Click "Most Recently Updated/Listed". Row renders bold + red + 6px red dot indicator. Other rows remain default.
  • 4 · Applied — Popup closes (via X, click outside, or row click). Trigger expands to show active label + inline red dot. Re-clicking trigger reopens at state 2.

Governance — Component classification

  • REUSE — Top bar (12322:72881), Side nav (12322:72859), Content area (12322:72897), Filter button, Sort trigger button, Save Search button, Tender card. All unchanged.
  • EXTENSION — Sort By popup (12837:20402). One new row at position 1; same row typography, padding, hover, and selected treatments as the existing three rows. No new visual language introduced.
  • NET-NEW — None.

Responsive behaviour

  • Desktop 1280px — Full layout per Figma source. Sort popup 230px wide, anchored to the left edge of the trigger.
  • Tablet 768px — Hide "Trades" column and result count. Reduce content padding to 16px. Sort popup unchanged.
  • Mobile 375px — Hide side nav (assumes hamburger pattern from app shell). Toolbar wraps to two rows. Hide table column header row; cards become stacked. Sort popup anchors to the right edge of the trigger so it stays on-screen.

Accessibility

  • All triggers are native <button> with aria-haspopup + aria-expanded.
  • Popup uses role="menu"; each row is a role="menuitem" with the selected row carrying aria-checked="true".
  • Focus rings visible on all interactive elements (2px outline #ff0000, 2px offset).
  • Selected state combines colour + weight + dot — meets WCAG 1.4.1 (colour is not the sole indicator).
