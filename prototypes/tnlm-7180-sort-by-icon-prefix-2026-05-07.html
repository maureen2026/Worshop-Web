Interaction States EXTENSION

breeze://design/components/TNLMSelect@current — new icon-prefix option variant. All five required states from the Action Items table are shown below as static frames.

1 — Closed, Default (no sort active)

Trigger shows "Sort by" placeholder. No icon in trigger. 1px solid --light-gray border. Chevron at 0°. No active indicator.

2 — Open, Idle (all options visible)
  • Move to Folder
  • Bid Date
  • Bid Value
  • Project Name (A–Z)

Border shifts to --teal. Chevron rotates 180°. Reserved leading slot (16px) visible on all rows. Icon (--medium-gray) on "Project Name (A–Z)" only.

3 — Open, New Option Hovered
  • Move to Folder
  • Bid Date
  • Bid Value
  • Project Name (A–Z)

Full option row bg: rgba(3,152,158,0.08). Icon colour shifts to --teal. Hover behaviour identical to existing options.

4 — Open, New Option Focused (keyboard)
  • Move to Folder
  • Bid Date
  • Bid Value
  • Project Name (A–Z)

Focus ring: outline: 2px solid var(--teal); outline-offset: -2px. Icon within focus boundary. Screen reader announces label only (icon aria-hidden).

5 — Closed, Active Sort (selected)

Trigger: border: 2px solid --teal. Label text weight 500 (--dark-gray). No icon in trigger — label-only keeps all four active-sort triggers consistent.

5b — Re-opened, Active Sort Marked
  • Move to Folder
  • Bid Date
  • Bid Value
  • Project Name (A–Z)

Active option: border-left: 2px solid --teal, bg rgba(3,152,158,0.08), trailing checkmark, icon shifts to --teal.

Accessibility — WCAG 2.1 AA

Keyboard & Screen Reader

  • Icon slot: aria-hidden="true" — purely decorative; label communicates sort type and direction
  • Trigger: role="button", aria-haspopup="listbox", aria-expanded toggles true/false
  • Panel: role="listbox"; each option: role="option", aria-selected
  • Arrow Down/Up cycles all four options; Enter selects; Escape closes and returns focus to trigger
  • Focus ring: 2px solid --teal, outline-offset -2px (inset) — no clipping at panel edge
  • Touch targets: option rows 40px height + 8px vertical padding = 44px minimum hit area
  • Content team to confirm: screen reader label "Project Name A to Z" vs. "Project Name (A–Z)" — open decision
  • Text contrast: --dark-gray (#454550) on --surface (#FFFFFF) = 8.6:1 (AAA)
  • Icon contrast on hover: --teal (#03989E) on --teal-hover-bg = meets 3:1 UI component threshold

Open Decisions — Maureen to Confirm

Decision 1 — Icon token

This spec renders an inline SVG representing a sort-A–Z pattern (ArrowDownAZ from Shadcn UI kit). Maureen must confirm which canonical icon token from the TNLM icon set maps to this visual. This is a hard blocker for FE-01 handoff.

Decision 2 — Reserved leading slot vs. flush alignment

This spec uses the reserved leading slot approach (Figma Recommended, node 143:248): all four option labels share the same left edge. The alternative — flush alignment where icon-free options start at x=16 — causes a 22px label shift that looks like a broken layout. Recommendation: reserved slot. Maureen to confirm.

Decision 3 — Icon in closed trigger when active

This spec shows label only ("Project Name (A–Z)") in the closed/active trigger. No icon appears in the collapsed state, keeping all four active-sort triggers visually consistent. Maureen to confirm this is the permanent design choice (not a temporary rollout differentiator).

Design System Review Required

The icon-prefix option variant is new to the governed TNLMSelect molecule. A brief async review with the Design System owner is required before this spec is sent to FE-01. This variant sets a precedent for all future icon-in-option uses across the product.
