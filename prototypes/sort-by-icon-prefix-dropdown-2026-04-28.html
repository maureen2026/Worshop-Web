Open Decisions — Maureen to Confirm

Decision 1 — Icon token

This spec uses an inline SVG representing a sort-ascending / A–Z icon (lines narrowing upward with a downward arrow). Confirm: Is this the correct icon from the TNLM icon set, or should a specific token (e.g. sort-alpha-asc, az-sort) be used instead? This is a blocker for FE-01.

Decision 2 — Reserved leading slot vs. flush alignment

This spec uses the reserved leading slot approach — all four option labels are left-aligned at the same horizontal position. The 16px slot is empty on the three icon-free options. This is the recommended approach. Approve or override to flush alignment?

Decision 3 — Icon in closed trigger when active

This spec shows label only in the closed/active trigger ("Project Name (A–Z)" — no icon prefix in the trigger button). This keeps the trigger consistent with how the existing three options appear when selected. Approve or override to show the icon in the trigger as well?

Design System Review Required

The icon-prefix option variant is new to the governed TNLMSelect molecule. Per the brief, an async check with the Design System owner is recommended before this spec is finalised — this sets the precedent for all future icon-in-option uses across LeadManager.
