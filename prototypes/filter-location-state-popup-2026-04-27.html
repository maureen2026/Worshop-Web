Keyboard Navigation & Accessibility Annotations
Keyboard
  • Tab — Move to filter button; when list open: move through list items
  • Enter / Space — Toggle filter open/close; select focused item
  • ↓ ↑ — Move through list items when open
  • Esc — Close list; focus returns to filter button
  • Home / End — Jump to first/last list item
ARIA & Screen Reader
  • role="button" aria-expanded="false|true" on filter button
  • role="listbox" on state list container
  • role="option" aria-selected="true|false" on each state item
  • aria-label="1 location selected" on badge
  • Focus management: on open → search input. On select → item. On Esc → filter button.
Scroll & Layout Specs
State list: max-height: 240px, overflow-y: auto, custom scrollbar (4px, teal-adjacent). Row height: 40px desktop / 48px tablet. Divider: 1px var(--light-gray) between rows. Search input auto-focused on list open. Clear button: bottom-right of list footer, disabled when no selection, enabled when selected.
Open Decision — Confirm Before FE Handoff
Single-select vs multi-select: This design uses radio (single-select). If confirmed as multi-select, swap StateListItem variant to "checkbox" — same component, same layout, no structural change. Confirm with frontend team (TNLM-7168 FE-01).
