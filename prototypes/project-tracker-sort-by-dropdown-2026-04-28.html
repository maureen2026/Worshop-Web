Open Decisions — Maureen to Confirm

Decision 1 — Icon identity (BLOCKER for FE-01)

This prototype uses a sort-ascending/lines icon as a placeholder. Maureen must confirm the canonical icon token from the TNLM icon set before FE-01 can proceed. Recommendation: a "sort A–Z" or "text sort" icon. Confirm that the chosen icon is not already in use for another meaning in the product.

Decision 2 — Reserved leading slot vs. flush alignment

This prototype implements the reserved leading slot approach — all four option labels align at the same left edge. The alternative (flush) aligns icon-free options to the panel left edge; the new option shifts right by 22px. Recommendation: reserved slot. Approve or override.

Decision 3 — Icon in collapsed trigger when selected

This prototype implements label-only in trigger when "Project Name (A–Z)" is the active sort (Frame 5). No icon appears in the closed trigger, keeping it consistent with the other three sort options. Approve or override.

Design System Review — async check recommended

This is the first icon-prefix variant in TNLMSelect. A brief review with the DS owner sets the precedent for all future icon-in-option uses. No ticket exists yet — Maureen to action.
